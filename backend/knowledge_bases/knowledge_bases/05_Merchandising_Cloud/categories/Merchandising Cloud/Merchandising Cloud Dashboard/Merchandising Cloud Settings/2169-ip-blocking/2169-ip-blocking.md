# IP Blocking

You can exclude specific IP addresses from your Merchandising Cloud analytics and personalization data to ensure your metrics accurately reflect real customer behavior rather than internal team activity. 

This capability helps you maintain cleaner analytics data and more accurate personalization algorithms by filtering out employee browsing patterns that could skew your merchandising insights.

Excluding internal IP addresses from analytics tracking enables more reliable data-driven merchandising decisions. When employees regularly visit your store to check products or test features, their behavior can affect personalization recommendations and analytics reporting. IP blocking ensures your performance metrics and AI-powered personalization reflect genuine customer patterns, leading to better optimization decisions.

When to Exclude IP Addresses

![574a7ae-Blog_Graphic_IP_Exclusion_V6.png](https://us.v-cdn.net/6038474/uploads/IYRBCTVEBNZ0/574a7ae-blog-graphic-ip-exclusion-v6.png)
As part of our ongoing quest to make merchant lives easier, an IP Exclusion capability has been released.

Clients have the ability to dictate which IP addresses to exclude from their Maropost Merchandising Cloud (formerly Findify) analytics and personalization data.

There are many reasons why a client would want to exclude certain IP addresses, including the common scenario where employees are regularly visiting the web store to check the products.

If unchecked, this behaviour can end up affecting the personalization algorithm as well as slightly skewing the analytics merchants rely on to make data-driven merchandising decisions.

![308f0fc-IP_Blocking_image.png](https://us.v-cdn.net/6038474/uploads/EZYA2CVAB0Y5/308f0fc-ip-blocking-image.png)

How to Exclude IP Addresses

Clients have only to log in to their merchant dashboard, then navigate first to ‘Settings’ then to ‘Advanced Setup’. Here you will find ‘IP Blocking’. In seconds, clients can easily add as many IP addresses as they wish.
