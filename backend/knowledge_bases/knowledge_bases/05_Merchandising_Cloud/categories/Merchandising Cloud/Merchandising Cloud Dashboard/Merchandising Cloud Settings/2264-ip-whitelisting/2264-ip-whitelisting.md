# IP Whitelisting

Implementing IP whitelisting helps you maintain secure development and staging environments while protecting your configuration changes. When you can restrict access to specific IP addresses, you help ensure that only authorized users can access sensitive setup areas, which supports better security practices and protects your merchandising configuration from unauthorized changes.

IP-based access control enables safer development workflows and testing procedures. This helps protect your ecommerce operations by ensuring configuration changes are made only by authorized team members, supporting better operational security and reducing risks from unauthorized access to your merchandising platform settings.

Product Sync Whitelisting

If accessing the product feed requires IP whitelisting in your firewall, you need to whitelist the following IPs for our product sync mechanism to download the product feed from you

- 34.69.253.49
- 34.136.57.29
- 34.70.45.199

Test environment access

If your test environment is hidden to the general public, but Maropost Merchandising Cloud (formerly Findify) requires access to it (to verify installation, do customizations or help with your queries), you can whitelist the following IP address and our team would be able to access your test environment:

- 50.16.12.102
