# Webstore Settings

Control the look and functionality of your Neto webstore with the webstore settings. Settings can be found in the **Webstore** > **Webstore Settings** menu.

| Name | Description |
| --- | --- |
| Site template set | Your website theme name. All website pages, templates and emails will be drawn from these folders in your site files. |
| Content type that represents product category menu | Select a content type to be used as your product category menu. |
| Show E-commerce Software branding | Display or hide the “Powered By Neto” branding on your site. |
| Default wish list name | Each customer on your site will have a wishlist with this name. |
| Auto approve new product reviews | If enabled all new product reviews will be automatically approved otherwise an administrator will need to manually approve the reviews. |
| Enable anonymous product reviews | Allows non registered users to post product reviews. |
| Only show in stock product in thumb list | If enabled items that are out of stock will not be shown on category pages and in product lists on your website. |
| Do not show product page if in compat category | Do not show the product display page if the product is assigned to a compatibility category. |
| Default sort order for product listings | Default sort order for product thumbnails on category pages on your website. Options: SKU, name, popular, highest_price, lowest_price. |
| Show out of stock product at last | Show products that are out of stock at the end of thumb lists. |
| Default thumbnail layout for product listings | Default thumbnail layout for product category pages on your website. Options: gallery, list or custom. |
| Maximum number of items per category page | The maximum number of items displayed per category page. Paging will come into effect when the total number of items in a category exceeds this amount. |
| Show lowest multi-level price as selling price on website | If enabled lowest multi-level price will show as the selling price on website. |
| Show products with no pricing | If enabled products with no pricing or zero in the price field will display on your web store. |
| Display upsell page before checkout | Display upsell page before checkout if an item has upsell products set up. |
| Maximum number of items in upsell list | The maximum number of upsell products displayed in the upsell list. If set to ‘0’ upsell products will not display. |
| Send restock report notification | If enabled will send you an email of products to be restocked. |
| Send stock warning notifications | If enabled will send you an email of products where stock level is low. |

## Default Sort Order for Product Listings

The table below lists the sort order options for products and how they are determined.

| Sort Order | Will sort |
| --- | --- |
| Default Sorting | Numerical sort order set on individual products |
| Product Name | Alphabetically by the Products Name |
| Product SKU | Alphabetically by the Products SKU |
| Product Brand | Alphabetically by the Products Brand |
| Lowest Price | Lowest price to highest price |
| Highest Price | Highest price to lowest price |
| Largest Item | Descending by the Largest Item (weight) |
| Smallest Item | Descending by the Smallest Item (weight) |
| Longest Item | Descending by the Longest Item (length) |
| Shortest Item | Descending by the Shortest Item (length) |
| Arrival Date | Descending by the arrival date of the Product |
| Set in Template | According to the pages template |
| Customer Wishlist | Descending by added date in Customer’s Wishlist |
| Most Popular | Descending by Most Popular Products |
| Sort Order 1 | Custom per product Sort Order 1 field |
| Sort Order 1 > Lowest Price | Custom per product Sort Order 1 field then Lowest Price |
| Sort Order 2 | Custom per product Sort Order 2 field |
| Sort Order 2 > Lowest Price | Custom per product Sort Order 1 field then Lowest Price |
| Sort Order 1 > Sort Order 2 | Custom per product Sort Order 1 then Sort Order 2 field |
| Product Type > Sort Order 2 | Product type is no longer in use, this will function as Custom per product Sort Order 2 field |
| Other | Not in use |

Please Note: When organising your products, **Sort Order** values work in ascending order, ie. 1 will be first and 100 will be last.
