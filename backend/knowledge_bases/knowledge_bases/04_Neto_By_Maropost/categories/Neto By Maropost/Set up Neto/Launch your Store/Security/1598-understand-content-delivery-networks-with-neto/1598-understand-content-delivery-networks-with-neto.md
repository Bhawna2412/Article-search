# Understand Content Delivery Networks with Neto

All Neto websites are routed through a global industrial CDN (Content Delivery Network) in order to serve website content to customers with high availability and high performance, whilst also maintaining high security for website and application data. Our CDN is a globally distributed system of data centres that use intelligent caching and cache control options, as well as optimisation tools for content and network to make websites and web applications run faster. To minimise the distance between the visitors and your website's server, a CDN stores a cached version of its content in multiple geographical locations.

For a CDN to work effectively, it needs to be the default inbound gateway for all incoming traffic. Therefore, it is Maropost company policy that we cannot support the use of multiple CDN accounts to route your website through. Seeing as all Neto websites are already configured through our CDN provider, adding an additional layer will cause issues with performance, caching and speed and may limit some security protocols.

For these reasons, Neto cannot provide support for site performance, site security, firewalls, web crawlers and/or bot attacks and various other security and performance aspects if you choose to use a 3rd-party CDN.
