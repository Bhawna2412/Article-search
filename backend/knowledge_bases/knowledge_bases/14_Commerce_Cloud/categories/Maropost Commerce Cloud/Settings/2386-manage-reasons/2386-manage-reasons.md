# Manage Reasons

The **Manage Reasons** settings allow you to create and maintain a standardized set of reasons that can be selected when processing **returns, refunds, and cancellations**. Instead of relying on free-text inputs, these predefined reasons ensure that post-order actions are handled in a consistent and structured way across your store.

By using standardized reasons, internal teams can quickly understand why an order action was taken, reduce ambiguity during order management, and maintain uniform reporting. 

This also improves customer communication by ensuring that clear, predefined explanations are used when orders are returned, refunded, or cancelled.

### To access

Go to **Back Office → Settings → Manage Reasons**

![image.png](https://us.v-cdn.net/6038474/uploads/AR1M96MLB2T9/image.png)

## Reasons Dashboard

When you open **Manage Reasons**, you will see a list of all reasons that currently exist in the system. This page provides a centralized view to review and manage your reasons.

![image.png](https://us.v-cdn.net/6038474/uploads/N53VV6JHOO1G/image.png)
The dashboard displays the following information for each reason:

- **Reason Label** – The name shown internally (for example, *Missing Items* or *Poor Quality*).
- **Reason Value** – The underlying value stored and used by the system.
- **Reason Type** – Indicates where the reason can be used:  - Return
  - Refund
  - Cancellation
- **Date Added** – When the reason was created.
- **Updated At** – When the reason was last modified.
- **Actions** – Options to manage the reason:  - **Edit** – Update the reason details
  - **Duplicate** – Create a copy of the reason
  - **Delete** – Remove the reason from the system

This page helps you quickly review existing reasons and maintain a clean, consistent list.

## Create a New Reason

To add a new reason, click **New Reason** from the top of the Reasons page.

![image.png](https://us.v-cdn.net/6038474/uploads/28NWCJV37AX0/image.png)
In the **Create Reason** panel, enter the following details:

- **Label**
A clear, user-friendly name for the reason.
- **Value**
The system value associated with the reason. This is required and should be unique.
- **Reason Type**
Select where the reason will be used:  - Return
  - Refund
  - Cancellation
- **Description**
Optional additional context to explain when or why the reason should be used.
  ![image.png](https://us.v-cdn.net/6038474/uploads/FZZ0WVPWDZR4/image.png)
- Once all required fields are completed, click **Save**.
  ![image.png](https://us.v-cdn.net/6038474/uploads/YXL899UZIOFP/image.png)
- The new reason will appear in the Reasons list and be available for use in the selected workflow.
  ![image.png](https://us.v-cdn.net/6038474/uploads/X1XKC40IHX95/image.png)
