# Release Notes v3.5.5 - POS and Back Office

**Release version:** 3.5.5 | **Release Date:** 6 June 2019

In this [update to v3.5.0](https://galaxy.maropost.com/s/article/Release-Notes-v3-5-0-Back-Office-and-PO) we've implemented new functionality with Offline POS and addressed **1 issue**. See below for more detail.

### **New Feature - Manually prompt a resync of Offline POS**

You can now force a resynchronisation of activity performed while in Offline POS mode!

**At a glance...**

- You can **manually****force an Offline POS synchronisation**
- Retail Express will try to synchronise transactions when your system returns online (as per the normal process)
- Forcing a synchronisation will **synchronise everything**

![mceclip1 - 2023-07-09T221925.485.png](https://us.v-cdn.net/6038474/uploads/attachments/migrated/image.05T5g00000ppfZeEAI.jpg)
With the new feature you will be able to manually prompt a resynchronisation of the Offline POS system. This will be useful in times where synchronisations were unable to be completed, or there's a need to resynchronise transactions.

To access the new feature:

1. Log into the Retail Express **Back Office**
2. Navigate to **Reports > External Services > Offline POS Servers**
3. The Resync button will appear in the new Resync column on the right

For more information on Offline POS, please see [Offline POS - Overview, Installation and FAQs](https://galaxy.maropost.com/s/article/Installing-Offline-POS).
