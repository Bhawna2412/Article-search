# Release Notes 3.12.0 - API Updates

In this release we have expanded the ability to manage Purchase Orders via the Inventory Planning API. For more information on changes in version 3.12.0 please refer to the [POS and Back Office Release Notes](https://galaxy.maropost.com/s/article/Release-Notes-3-12-0-Special-Orders-are-here).

|  |  |
| --- | --- |
| **API** | **Details** |
| Inventory Planning API | Using CreateUpdatePurchaseOrders API method the Outlet on a Purchase Order can now be changed if the Purchase Order is in an Incomplete state.  All other states will return the following error:  The Outlet for a Purchase Order cannot be changed |
| eCommerce API | The release v3.12.0 introduced the Special Orders feature. Please note that any quantities or ETA calculations from Purchase Orders will exclude stock linked to a Special Order item. |
