# Release Notes v3.13.6 - POS and Back Office

**Release Date:** 17th May 2021

In this update we've added some highly anticipated features, including some exciting new Shopify additions, Promotional Campaign features and more! Read on for the summary, or take a look at our [Announcements](http://galaxy.maropost.com/home/leaving?allowTrusted=1&target=https%3A%2F%2Fress.zendesk.com%2Fhc%2Fen-us%2Fsections%2F200578310) section for detailed information, including examples on how to use the new features!

**New:**

* New [**Promotional Campaign**](https://galaxy.maropost.com/s/article/Announcing-new-Promotional-Campaign-options) options:
  + "Buys at least" - Qualifying Type
  + "$ unit price" - Discount Option
* **Shopify**:
  + [**Split Fulfilment**](https://galaxy.maropost.com/s/article/Announcing-Shopify-Split-Web-Order-Fulfilment) - Ability to automatically split the fulfilment location of web orders if the order cannot be fulfilled from a single outlet, saving time and shipping costs!
  + [**Source and Fulfil Location Optimisations**](https://galaxy.maropost.com/s/article/Announcing-optimised-Shopify-Source-and-Fulfilment) - we've optimised the Source From/Fulfil From Outlet selection for more intelligent inventory control
  + [**Post Codes**](https://galaxy.maropost.com/s/article/Post-Code-Ranges-for-Shopify-Fulfilment-Groups) - We've added support for Post Code ranges in Fulfilment Groups for greater fine tuning of your source and fulfil locations for customer orders
* [**POS Alert Configuration**](https://galaxy.maropost.com/s/article/New-POS-Web-Order-Alert-configuration-options) - Added new a option to POS Order Alert settings to separate Click & Collect from the count of Web Orders (making it easier for staff to see how many orders require action)

**Updated:**

* Resolved issue with Browser Stocktake where count was not increased for products with more than 2 additional barcodes
* Shopify Integration - Disabled ability to reorder Inventory Locations when Fulfilment Locations is enabled
