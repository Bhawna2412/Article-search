# Release Notes 3.15.7 - POS and Back Office

# Release Notes 3.15.7 - POS and Back Office

This update will become available to all users by 12th July 2022.

## New Features

### New Payment Types

We've added some new [Payment Types](https://galaxy.maropost.com/s/article/Configuring-Payment-Types) to Retail Express, providing users with more detailed financial reporting and reconciliation.

These Payment Types can be manually selected in POS or mapped to online Payment Methods for eCommerce Integrations.

* Myer
* Westfield Direct
* BPAY
* Klarna
* DoorDash
* Latitude
* Internal Voucher
* Buy Aussie Now
* Trade In

The new [Payment Types](https://galaxy.maropost.com/s/article/Configuring-Payment-Types) are available via **Settings > Payment Types** and will be disabled by default.

## Improvements and Resolutions

### Improvements

* The POS Email Template placeholder [Customer Name] will now display the customer's First and Last Name (the [Customer First Name] field is still available to display the first name only).

### Resolutions

* Customer details will now be saved if there is a full stop (.) in their name. Additionally, any invalid characters in their name that cannot be saved will prompt an error when the user tries to save the customer in Back Office.
* The Back Office Dashboard "Staff" tab now displays data as per the selected Date Range filter
