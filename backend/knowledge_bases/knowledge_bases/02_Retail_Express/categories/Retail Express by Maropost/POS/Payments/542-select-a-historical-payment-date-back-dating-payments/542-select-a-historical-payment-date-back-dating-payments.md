# Select a Historical Payment Date (Back-dating Payments)

When adding Payments to a sale you can select a historical date for the payment, generally required when entering direct deposits. This is done using the Edit Payment screen.

## Before you Begin

- A User must have sufficient POS Security Features enabled to be able to select a historical date for a sale
- Back-dating must be done *prior* to the payment being processed/finalised. You cannot back-date a finalised payment (you will need to reverse the payment)
- You can also back-date a payment reversal that has not yet been processed/finalised (see FAQs)

## Back-dating payments

To back-date a payment:

1. Log into** POS**
2. Create a sale (or open an existing sale)
3. On the **Payments Tab** click to select the Payment method
4. Enter the **Payment Value** (as per the normal process)
  ![User-added image](https://us.v-cdn.net/6038474/uploads/attachments/migrated/Screenshot20241203at32554AM.05TJ100000OOLJhMAP.png)
5. **Double click the payment** to open the edit payment screen (or press and hold for touch screens/mobile devices)
  You can only do this if the payment has not yet been finalised (and you can still see the trash can icon next to the payment)

  ![User-added image](https://us.v-cdn.net/6038474/uploads/attachments/migrated/Screenshot20241203at32720AM.05TJ100000OOLL4MAP.png)
6. Select the **Payment Date**
  ![User-added image](https://us.v-cdn.net/6038474/uploads/attachments/migrated/Screenshot20241203at32759AM.05TJ100000OOLLJMA5.png)
7. Click **Update Payment**
8. Click **Update** to finalise the sale and Payment
