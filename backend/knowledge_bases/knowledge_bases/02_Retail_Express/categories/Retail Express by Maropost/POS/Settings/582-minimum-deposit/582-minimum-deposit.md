# Minimum Deposit

You can configure minimum deposit requirements in Retail Express for special orders and laybys, helping your retail operations secure customer commitment while managing inventory allocation effectively. This capability ensures customers provide adequate upfront payment before you allocate inventory or place special orders, which helps reduce order cancellations and protect against financial loss.

Setting up appropriate deposit thresholds enables your retail operations to balance customer accessibility with financial security, helping optimize special order management. This helps retail businesses maintain healthy cash flow while accommodating customer payment preferences for higher-value purchases and custom orders.

Within POS you can control whether customers will need to pay a minimum deposit to finalise the sale. If using the Special Orders feature you can also configure a separate Minimum Deposit rule for Special Order items.

Minimum Deposits can be entered as a dollar value, or as a percentage of the sale.

All fulfilled products will need to be paid in full before the sale can be finalised.

## Configuring the Minimum Deposit

To configure the Minimum Deposit:

1. Navigate to **Settings > POS Settings > Order Settings**
2. Enable the setting **Allow Finalise When Minimum Deposit Not Met**
3. Enter a value into the **Minimum Deposit **field

> **Note:** Enter either a percentage or dollar value

  ![mceclip2 (42).png](https://us.v-cdn.net/6038474/uploads/attachments/migrated/image.05T5g00000ppVFXEA2.jpg)
4. Enter a **Special Orders ****Minimum Deposit **value if using Special Orders
5. Click **Save Changes**
6. Navigate to **Staff > POS Security Features**
7. Select a **Profile **from the drop-down
8. Enable the setting **Allow Override of Minimum Deposit**
  ![mceclip3 (28).png](https://us.v-cdn.net/6038474/uploads/attachments/migrated/image.05T5g00000ppVFcEAM.jpg)
9. Click **Save**

## Overriding the Minimum Deposit

To override the Minimum Deposit requirement you will need to enable the "Allow Finalise when Minimum Deposit not met" setting in the Security Profile so that sales can still be created.

If the user also has the POS Security Feature for "Allow Override of Minimum Deposit" the sale will be created without incident. If this permission has not been granted, a warning will be displayed to the user, and they will be prompted to enter the credentials of a user with access.
