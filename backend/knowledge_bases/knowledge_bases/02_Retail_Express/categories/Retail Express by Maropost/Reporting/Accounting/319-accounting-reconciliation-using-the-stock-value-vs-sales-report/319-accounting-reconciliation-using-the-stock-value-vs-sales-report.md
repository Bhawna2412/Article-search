# Accounting Reconciliation using the Stock Value vs Sales Report

You can perform monthly accounting reconciliation by using the Stock Value vs Sales Report to verify that inventory movements align with sales transactions and cost of goods sold calculations. This reconciliation process helps retail operators ensure financial accuracy, identify accounting discrepancies, and maintain proper inventory valuation for accurate financial statements and tax reporting.

Using the Stock Value vs Sales Report for accounting reconciliation helps retail operators maintain financial integrity, support audit requirements, and ensure inventory records match financial records for reliable business reporting.

The Stock Value vs Sales Report is one of the most popular reports in Retail Express and includes extensive information on the current inventory value. This makes it an ideal report to use to reconcile to an Inventory account for accounting purposes.

> **Note:** For detailed information on using the report refer to the Reports - Stock Value vs Sales Report article.

> **Note:** Data presented in the Stock Value vs Sales report is current at the time of running the report. This report is unable to provide data from a historical point in time.

It is advised to run this report after the close of business on the final day of each month, then Export the results to Excel to save a specific point in time.

## Using the Stock Value vs Sales Report

1. Navigate to **Reports > Stock Reports > Stock Value vs Sales**
2. Click **Refine Search**
3. Select the following **Stock Value Based On **fields:
  ![ARSVVSR_StockValueBasedOn.jpg](https://us.v-cdn.net/6038474/uploads/attachments/migrated/image.05T5g00000pmCx3EAE.jpg)

> **Note:** Hold Ctrl to select multiple options

4. Click **Search**
5. The **Current Value Buy Ex** column in the **Current Stock **section should match the value in your Inventory Account.

> **Note:** If you're using the Retail Express Xero or MYOB integrations with Direct Cost provisioning you can compare the column Current Value Direct Costs to your Direct Cost Provisioning Account. If you're using the same account for both Inventory and Direct Cost Provisioning you can use the Current Value COGS column.

## []()Frequently Asked Questions

[My Inventory Account doesn't match the report](https://galaxy.maropost.com/kb/articles/319-accounting-reconciliation-using-the-stock-value-vs-sales-report#no-match)

If the amount in the Stock Value vs Sales report doesn't match your accounting package this could be due to:

- An **Initial Reconciliation **not being done before beginning the integration
- **Manual changes **to the **Buy Price **in Retail Express
- **Other Transactions **posted to the **Inventory Account **that didn't originate from Retail Express
