# Stocktake Report

You can generate stocktake reports to review count results, analyze variances, and document inventory adjustments, helping maintain audit trails and identify patterns in inventory discrepancies. This improves your retail operations by providing comprehensive stocktake documentation, with properly configured stocktake reporting typically enabling better shrinkage analysis and inventory control improvements.

Understanding stocktake reporting involves recognizing variance metrics, adjustment documentation, and trend analysis capabilities to help retail operators focus on the right inventory accuracy analytics workflows, ensuring your retail strategy consistently drives better stock control and loss prevention outcomes.

The Stock Take Report and Stock Take Report (Limited) are designed to report on completed Stocktakes, and if using the Browser Stocktake method only those that have been Authorised.

Both reports access the same information and allow the same search options however the Limited report displays basic product information and count quantities without displaying cost-related information (Buy Ex and Variance).

To control which version of the report the User can access, you will need to modify their associated Security Profile settings.

## []()Using the Stocktake Report

To access the Stock Take Report:

1. Navigate to **Reports  > Stock Reports > Stocktake Report** or **Reports > Stock Reports > Stocktake Report (Limited)**
2. Select from the **Filters** as required (see below for detailed information)
3. Click **Search**
4. Click **View Details **against a corresponding Stocktake to view detailed information (including product breakdowns etc)
5. Click **Print **to print a copy of the report

## []()Filters

|  |  |
| --- | --- |
| **Filter** | **Details** |
| Order By | Selects the display order. Options are:  - Outlet - Date - Sales Person  For example, you may choose to sort by Date to view the oldest Stocktakes first |
| Status | The Inventory Status that was updated at the time of doing the stocktake e.g. Available (for Available stock only) or Stock on Hand (includes Allocated Stock - refer to the Stocktake articles for more information) |
| From and To | The date range for when the Stocktake was Authorised/Uploaded (not the date the Stocktake was created) |
| Sales Person | Used to filter Stocktakes from specific user/s. The first field allows you to filter by All Users, Active Users Only or Inactive Users only. The second field will update the results based on this selection.  > **Note:** Hold the Ctrl button on the keyboard to select multiple Users |
| Product Type | Used to filter Stocktakes for specific Product Types only |
| Outlet | Which Outlet (Store Location or Warehouse) the Stocktake was conducted for.  > **Note:** Only the Outlets the current user has access to will be displayed |

## []()Results

The Results tab displays a summary of your Stocktake information. Click **View Details** to view more detailed information about an individual Stocktake.

|  |  |
| --- | --- |
| **Column** | **Details** |
| Details | Link to view the Stock Take Details Report (also accessed via **Reports > Stock Reports > Stocktake Detail Report**) |
| Title | The name of the Stocktake. Browser Stocktakes can have a name entered, otherwise it will display the name of the file uploaded to Retail Express |
| Date | The date of the Stocktake |
| Warehouse | The Outlet the Stocktake was performed for |
| Status | The Stock Take Method selected at the time of creating the Stocktake  - AVL - Adjust Available - SOH - Adjust Stock On Hand |
| Sales Rep | The user that performed the Stocktake |
| Buy Ex Variance | Using the quantity variance (the difference between the number of expected products in the Stocktake vs the number of products actually counted), this displays the total value of the variance using the Outlet-specific Buy Ex price. The outlet used is the Outlet where the Stocktake was performed. |
| COGS Variance | Using the quantity variance (the difference between the number of expected products in the Stocktake vs the number of products actually counted), this displays the total value of the variance using the Outlet-specific Cost of Goods Sold (COGS). The outlet used is the Outlet where the Stocktake was performed. |
