# Marketing Cloud Release Notes – 13 May 2020

## Overview

In this release, we introduced a new reporting tool - Growth and Attrition Reports. In this release, we have changed how our application is updated to a newer version. With the application version update prompts, you have more control over the update process.

To know more, read on!

## New Features

### Growth and Attrition Report

The Growth and Attrition Report is a metrics report on user growth and attrition to help you understand how your contact lists are growing or shrinking. The new report is available at **Analytics** > **Custom Reports**.

![Custom_Reports_-_Growth_Attrition.png](https://us.v-cdn.net/6038474/uploads/attachments/migrated/05T5g00000BkWr9.05T5g00000BkWr9EAF.png)

While generating the report, you can choose the contact lists, date range, and metrics to be included in the report. Furthermore, you can specify the report output and delivery details. You can specify the schedule of the report generation and decide if you want a one-time report or a recurring report.

For detailed information, see [Custom Reports: Overview](https://galaxy.maropost.com/kb/articles/994-custom-reports-overview) and [Creating a Growth and Attrition Report](https://galaxy.maropost.com/kb/articles/837-creating-growth-attrition-reports).

### Application Version Update Prompt

Starting this release, we are introducing prompt-based application updates. What it means is that the application updates will not be initiated by us anymore! When a newer version is available, you will be prompted to update. If dismissed, you will be prompted again in three hours. You will keep seeing the prompt every three hours until the update is done.

> **Note:** To leverage all the new features and fixes, we encourage you to keep your application updated to the latest version.

Check out the FAQs, see [Application Version Update Prompts](https://galaxy.maropost.com/kb/articles/642-application-version-update-prompts).
