# Marketing Cloud Release Notes – 18 Jun 2024

### **OVERVIEW:**

Greetings to all our Maropostian users out there!!

In this release, we are expanding our abandoned cart feature integration beyond our existing Neto, Shopify, and ClickBank users by making our **REST API** payload public to other account holders as well.

**Note:** Only registered Marketing Cloud users can access the **REST API** payloads.

In addition, we have also made some layout and UI modifications to our Recurring Custom Email Campaign Reports.

To learn more about these features, read ahead!!

### **ABANDONED CART REST API FEATURE:**

Registered Marketing Cloud users can use this interface to authenticate and update their store details with us. You can use the **Authenticate and Update** **REST API** feature to integrate multiple store accounts and generate unique Auth Keys to grant multiple API requests.

After your integration setup, you can send your store’s abandoned cart data to us through the **Abandoned Cart** **Payload API**. In adherence to data privacy laws, only the customer ID, abandoned cart ID, and order details are stored with us.

With the help of the **REST API** feature, you can create an automated process for sending abandoned cart email messages to your contacts via journeys.

In addition, we have created a separate module for you to check your integrated stores and allow you to make any changes if necessary. You can access this section from **Navigation** >> **Integrations** >> **Abandoned Cart REST API**.

![abncart8](https://us.v-cdn.net/6038474/uploads/attachments/migrated/abncart8.068J1000001XFqpIAG.png)This API functionality can also be availed of by Neto, Shopify, and ClickBank users if they only wish to send abandoned cart-related campaigns.

To learn how to integrate your stores with your Marketing Cloud account via API requests, kindly read our [**Abandoned Cart REST API**](https://galaxy.maropost.com/s/article/Abandoned-Cart-REST-API-Integration) article.

### **RECURRING CUSTOM EMAIL REPORT ENHANCEMENT:**

With this revamp, the layout of our recurring reports is streamlined with an updated UI design, and the campaign types are combined in a single field. Users from now on can choose the campaign type from the list and select the brands, names, and campaign tags, respectively.

Additionally, you can include all the new campaigns from their report’s inception to their delivery date. This option is exclusive to recurring reports only.
