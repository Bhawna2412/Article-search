# Deploy - New Unified Core API

This article explains how to enable and deploy the new Unified Core API once accounts have confirmed that the client is being charged an ongoing monthly fee for API access:

Enable Unified/Core API on a Database

![rtaImage (50).png](https://us.v-cdn.net/6038474/uploads/attachments/migrated/image.05T5g00000poweXEAQ.jpg)![rtaImage (51).png](https://us.v-cdn.net/6038474/uploads/attachments/migrated/image.05T5g00000powecEAA.jpg)![rtaImage (52).png](https://us.v-cdn.net/6038474/uploads/attachments/migrated/image.05T5g00000powehEAA.jpg)![rtaImage (54).png](https://us.v-cdn.net/6038474/uploads/attachments/migrated/image.05T5g00000powemEAA.jpg)

1. Login to MGMT

2. Navigate to **Licenses > Add Client License** *(and wait for screen to refresh..)*

3. Select relevant Client Site from the list

4. Select **License Type: Core API** from dropdown

5. Ensure **Enabled:** checkbox is ticked

6. Click on **Save** button

7. Ensure that a success message is returned and that you can now see that database under the **Licenses > Client License List** in MGMT (which lists all databases that currently have the Core API enable)

Zendesk - Sending the email 

![rtaImage (53).png](https://us.v-cdn.net/6038474/uploads/attachments/migrated/image.05T5g00000powewEAA.jpg)

1. Add a new ticket

2. Add the email address for the contact on the solutions agreement

3. Select the macro **Finance > API > Unified API Deployment**

4. Replace *xxx.retailexpress.com.au* with the relevant database name

5. Submit the ticket as Solved
