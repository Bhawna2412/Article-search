# Sales: Daily Contractual related tasks

This article explains how to perform the daily tasks relating to recently created contracts.

Monitoring Panda Doc inbox

This is to maintain the incoming documents into Panda Docs and how to identify any that require action.

Navigate to **Documents > All**sort by modified date

Drafts > 1 day = DELETE

Expired = Delete

Signed = put in relevant sales person folders to file

Zoho CRM PandaDoc Dashboard

Closed Won = make sure in panda

[Contracts@retailexpress.com.au](mailto:Contracts@retailexpress.com.au)

Always 2 x emails successfully processed and actioned, if only 1 email arrives then there's an issue

Click on quotes and check the following:

subscription\_actioned (ticked)

redirect\_URL (send a brand new customer this if they close before giving payment details),

Invoice\_ID (Sub\_ID sometimes)

Books URL (links to invoice was to be paid via CC) else URL where they land for bank details

When this occurs the following steps need to performed to resolve the issue:

1. Select Account > Quote then click Zoho Finance and click New Invoice then close and that will force the customer to push over

2. Select Create Zoho Invoice (?)

2. Select Create Update Subs

3. Check Panda doc to see payment methods chosen and if Payment Plan (regardless of dd or cc) select Create Payment Plan

4. Create appropriate link(s) in Customer Control Centre > Client Administration > Customer Manager and send appropriate link(s) to client via email

Contract Signed but No Payment Details Entered

This is commonly caused by the client exiting out of the browser before entering details into the payment methods screen.

Client automatically gets an email 1 day later with the appropriate auto generated links to enter their details

Check this report on a daily basis to see which clients fall into this category
