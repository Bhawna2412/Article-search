# Support - Dev Liaison Process

# Support - Dev Liaison Process

The following article outlines the process used to escalate tickets to development through the Dev Liaison Reporting process.

In this article:

- [Process: Bugs/Suspected Bugs](#bugs)
- [Process: Data Changes - Inventory scripts, POS rounding statements etc](#data-changes)
- [Process: PEP - Product Enhancements and/or changes](#pep)
- [How to: Linking tickets](#linking)

**Please ensure all required fields (including the subject field) are completed.**

## []()**Bugs/Suspected Bugs**

For Bugs/Suspected Bugs please follow the below process:

1. Create a **new ticket**
2. Use the **Core Problem: ****Requires Investigation (New Ticket) **macro

> **Note:** this will no-longer be the customer-facing ticket

3. **Fill in all the fields** providing as much detail as possible, including examples where possible
4. Click **Submit**

> **Note:** The ticket will be assigned to Alan

5. **Link** the original customer ticket to the new ticket
6. **Reply to the customer**
7. Put the **original ticket on hold**

## []()**Data changes**

For data changes like requesting inventory scripts, or POS rounding statements etc, please follow the below process:

1. Create a **new ticket**
2. Use the **Core request: New Request (New Ticket)** macro
3. **Fill in all the fields** providing as much detail as possible, including examples where possible

> **Note:** For Inventory Scripts please include an example product with incorrect inventory so support/dev can review

4. **Submit **the new ticket
5. **Link **the original customer ticket to the new ticket
6. **Reply to the customer**
7. Put the **original ticket on hold**

## []()**PEP/Change Requests**

For product enhancement or change requests, please follow the below process:

1. Create a **new ticket**
2. Use the **Core PEP: New PEP (New Ticket)** macro
3. **Fill in all the fields** providing as much detail as possible

> **Note:** For PEP?s we need as much detail from the customers as possible including screenshots/mockups where applicable.

4. **Submit** the ticket.
5. **Link** the original customer ticket to the new ticket.
6. **Reply to the customer**
7. **Solve the original ticket**

## []()**How to: Linking Tickets**

Once the new development ticket has been created make a note of the ticket number.

1. On the customer facing ticket change the field **Type** to ?Incident?.
2. A new field will then appear just under this field called **Linked Problem**
  ![rtaImage - 2023-07-10T023321.058.png](https://us.v-cdn.net/6038474/uploads/attachments/migrated/image.05T5g00000ppiC3EAI.jpg)
3. Enter the **dev liaison ticket number**

> **Note:** you should be prompted with the dev ticket; this can sometimes take a few minutes to populate
