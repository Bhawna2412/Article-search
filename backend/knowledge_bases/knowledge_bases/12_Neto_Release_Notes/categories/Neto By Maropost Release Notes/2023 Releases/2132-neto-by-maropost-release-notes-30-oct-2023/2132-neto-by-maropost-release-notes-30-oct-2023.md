# Neto by Maropost Release Notes – 30 Oct 2023

## **Overview**

This release includes an important update for Catch integration.

## **Enhancements**

### **Mapping Available for the New Fields in Catch**

Catch recently introduced two new fields – **Shipped-from Country** and** Shipped-from Postcode** – to help increase the accuracy of the estimated delivery date. In Neto, these two new fields are now available to be mapped in your Catch integration.

To learn about Catch category and field mapping, see [**Set up Catch Category and Field Mapping**](https://galaxy.maropost.com/s/article/set-up-catch-category-and-field-mapping).

![NewFields](https://us.v-cdn.net/6038474/uploads/attachments/migrated/NewFields.0685g00000HhzAqAAJ.png)

To map the fields, in Navigation, go to **Catch** >> **Product Field Mapping**. Then, on the **Product Field Mapping** page, scroll down to the **Category specific product fields** section and find the new fields by clicking on the category that you wish to update.

> **Tip**: Click the information (i) icon to learn about the requirements and accepted values. For example, for the Shipped-from Country field, if all items within a category are shipped from the same country, select the "type in a value" option and enter a static value (say, AU) in the subsequent field. All acceptable static values are listed in the information tooltip.
