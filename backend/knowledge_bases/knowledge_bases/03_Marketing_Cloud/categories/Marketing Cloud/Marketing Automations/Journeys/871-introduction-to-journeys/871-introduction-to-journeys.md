# Introduction to Journeys

**Journeys** present a schematic for sending automated messages to individual contacts.

![Journey.png](https://us.v-cdn.net/6038474/uploads/attachments/migrated/05T5g00000BkWph.05T5g00000BkWphEAF.png)You can use the following widgets to create a Journey:

- **Journey Triggers:** Use the Journey Trigger widget to define the rules by which a contact enters a Journey. Every trigger has a different function and can be used to design a creative Journey that will engage your contacts and automate your email marketing process. [**Learn More**](https://galaxy.maropost.com/s/article/Journey-Triggers)
- **Journey Actions:**  Use a Journey Action widget to define what should be done for the contact inside the Journey. An email message can be sent, the contact’s profile can be updated, a contact tag can be assigned, or the subscription status of a list can be changed. [**Learn More**](https://galaxy.maropost.com/s/article/Journey-Actions)
- **Journey Filters:**  Use a Journey Filter widget to add branching logic to the Journey. You can have the contact flow into different branches depending on which segment it belongs to. You can also use filters to do split testing with A|B testing or Percentage Splits. [**Learn More**](https://galaxy.maropost.com/s/article/Journey-Filters)
- **Journey Delays:** Use a Journey Delay widget to inject pauses into the Journey. You can pause for a certain period or until a specific date and time. The most common use of Delays is for an onboarding series in which you want to send a series of emails to the contact, but put a pause in between each email. [**Learn More**](https://galaxy.maropost.com/s/article/Journey-Delays)
- **Journey End:** Use the Journey End widget to terminate a Journey. Every connected branch within a Journey must terminate with an End widget. [**Learn More**](https://galaxy.maropost.com/s/article/Journey-End)

**Note**: All actions, filters, and delays have two nodes each to accept input and give an output.

### Managing Journeys

You can create a new Journey and manage your existing Journeys on the Journeys index page. To access the page, in Navigation, go to AUTOMATION and then select Journey.

The following is a screenshot of the Journeys index page:

![Journey_Index_Page.png](https://us.v-cdn.net/6038474/uploads/attachments/migrated/05T5g00000BkWpi.05T5g00000BkWpiEAF.png)
