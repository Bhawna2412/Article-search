# Managing Users and Permissions in Marketing Cloud

You can configure user permissions to streamline your marketing operations while maintaining security and control over your Marketing Cloud account. This enables more efficient team collaboration by allowing you to delegate specific responsibilities to team members without compromising account security, helping optimize your workflow efficiency, and ensuring the right people have access to the tools they need to execute successful email campaigns.

When properly configured, user management helps reduce manual oversight while maintaining accountability across your marketing operations. This approach scales personalized access control as your team grows, ensuring both productivity and security standards are maintained throughout your email marketing programs.

Maropost has a dedicated section where you can add users and manage their permissions. Both admins and users (who are granted full access by the admin) can grant/revoke different levels of access to various Marketing Cloud features for other users.

Although a Marketing Cloud account has only one admin account, a user can assume roles similar to those of an admin, allowing numerous users to manage the account.

> **Note:** **Note:** Since we are introducing a **Unified Platform** (**Marketing + Service Cloud**) for our Maropost accounts, existing and new users who have purchased both Marketing and Service Cloud from us can manage their user privileges directly in the **Users & Permissions** module.

## Adding a New User

To add a new user to the Marketing Cloud Platform, follow the steps below:

1. Log in to your Marketing Cloud account, then click your profile drop-down menu and select **Users and Permissions**.

![User-added image](https://us.v-cdn.net/6038474/uploads/attachments/migrated/per0.05T5g00000tvQaTEAU.png)
2. The **Users and Permissions** page opens, displaying a list of all the existing users who have been given access by the admin to manage the features present on the platform. To add a new user, click on **Invite**.

![User-added image](https://us.v-cdn.net/6038474/uploads/attachments/migrated/per1.05T5g00000tvQaiEAE.png)
3. On the **Add User** slider, enter the user's email address since it is a mandatory process.

![User-added image](https://us.v-cdn.net/6038474/uploads/attachments/migrated/per2.05T5g00000tvQanEAE.png)
4. Permissions are categorized into **General, Marketing**, and **Service** fields. Under the **General** section, you can assign either a "**View Only**" or "**Full Access**" to a user.

![User-added image](https://us.v-cdn.net/6038474/uploads/attachments/migrated/per3.05T5g00000tvQb2EAE.png)

> **Note:** **Note:** Depending on your Maropost subscription, the **General, Marketing,** and **Service** fields may be enabled or disabled.

5. If you want a user to be a point of contact for communications from Maropost, set the user as a **Primary Contact**. 

![User-added image](https://us.v-cdn.net/6038474/uploads/attachments/migrated/per4.05T5g00000tvQcoEAE.png)

> **Note:** **Note:** A Marketing Cloud Platform can only have one primary contact per account.

6. Under the **Marketing** section, you can grant/revoke a user's permission to access the Marketing Cloud entirely by toggling the Access permission between **ON** and **OFF**.

![User-added image](https://us.v-cdn.net/6038474/uploads/attachments/migrated/per7.05T5g00000tv9BaEAI.png)
7. You can also assign specific roles to users in handling Marketing Cloud modules. You can provide full or partial access or even revoke access to sections such as Campaigns, Content, Reports, Audience, and so on.

![User-added image](https://us.v-cdn.net/6038474/uploads/attachments/migrated/per5.05T5g00000tvQiDEAU.png)
8. Authorization can be granted or revoked for the remaining Marketing Cloud modules. Save the changes that you have made.

![User-added image](https://us.v-cdn.net/6038474/uploads/attachments/migrated/per6.05T5g00000tvPZgEAM.png)
If you have purchased a Service Cloud package from Maropost, you can grant users full or partial access. To learn more about the various access controls present in a Service Cloud Platform, kindly read our [**Users and Permissions**](https://galaxy.maropost.com/s/article/Managing-Users-and-Permissions-in-Service-Cloud) page in the Service Cloud section.

![User-added image](https://us.v-cdn.net/6038474/uploads/attachments/migrated/per8.05T5g00000tvQlCEAU.png)

## Managing Users and Permissions

On the Index page, the user list is displayed as follows:

- **Email:** It contains the user's email address.
- **Access:** Displays which platform the user has access to.
- **Role:** Displays the status of an agent within the operations team. An agent can be an admin, a primary contact, or a user.
- **Actions:** This widget allows you to edit, activate/deactivate, duplicate, or delete a user from the list.

> **Note:** **Note:** Admins cannot be deleted from the list.

![activate0.png](https://us.v-cdn.net/6038474/uploads/7WB2JQT815N3/activate0.png)
To give administrators better control over access rights for third-party members or employees under contract, you can activate/deactivate user control as per your requirements.

![activate1.png](https://us.v-cdn.net/6038474/uploads/0AQ8JRDHT8HI/activate1.png)
Once deactivated, the employee's access to the application is revoked, and their name is grayed out in the users & permissions list.

To reactivate user access, administrators need to click the **Activate** option next to the grayed-out user name. Upon user reactivation, administrators must restore all previously assigned permissions.

![per14.png](https://us.v-cdn.net/6038474/uploads/ZSPAR19CH4JE/per14.png)

> **Note:** **Note:** Admins cannot be activated/deactivated from the list.

## Marketing Cloud User Privileges

### Campaigns

A user who has this permission can browse the campaign index page and create [**new campaigns**](https://galaxy.maropost.com/s/article/Creating-Email-Campaigns) and [**A/B Campaigns**](https://galaxy.maropost.com/s/article/Creating-AB-Campaigns). Existing campaigns (recurring, planned, and campaigns kept merely as drafts) can also be edited by the user. A campaign that has already been sent, however, cannot be changed or deleted by the user.

A user can view the reports and in-depth graphs specific to a campaign by using the campaign show page. However, they won't be able to see the ‘Overlay.'

Also, the admins can decide whether the user needs full access to the campaigns module or only limited control, depending on the roles assigned to them. To learn more about the access levels available in campaigns, visit and read our [**Access for Campaign creation**](https://galaxy.maropost.com/s/article/Restricted-Access-for-Campaign-Creation) page.

### Audience (contacts)

Users with full privileges for this section can access, manage, create, and export lists and segments. Some of the additional tasks are described below:

**Lists**

- Create new lists.
- Edit lists.
- Delete lists.
- Import contacts into lists.
- View contacts present in the lists.
- Delete and edit the contacts present in the lists.
- Access the contact show page from the lists.
- Refresh lists to see the latest count.

**Fields**

- Create new custom fields.
- Edit custom fields.
- Delete custom fields.
- Edit the default field value.

**Tags**

- Create new tags.
- Edit tags.
- Delete tags.
- Import contacts into tags.

**Brands**

- Add new brands.
- Edit brands.
- Access list details from within a brand.

**Segments**

- Create new segments.
- Edit segments.
- Delete segments.
- Duplicate segments.

### Survey

With this permission, the user can edit, preview, and delete existing surveys as well as create new ones. The web tracking field is visible to the user, but it won't appear until they have access to surveys or forms.

### Automation

With this privilege, the user can access workflows and cleansing rules.

### Forms

With the help of this permission, the user can access forms and various actions associated with them, such as:

- Create a new form.
- Preview forms.
- Check the HTML code for forms.
- Edit and delete forms.
- Export pending contacts from double-opt-in forms.

### Relational Tables

Users having privileges in this section can access Maropost relational tables and SQL queries.

**Tables**

- Create a new table.
- Edit existing tables.
- Delete and duplicate existing tables.
- Import contacts into tables.
- Export contacts from tables.

**SQL Queries**

- Create new SQL queries.
- Edit and delete existing ones.
- Export and execute.

### Mobile

The user has full access to all mobile-related features with this privilege.

### Push Notifications

With this privilege, the user can access the push notifications in their account.

### Coupon Codes

When admins grant users access to this section, they can create and manage coupon codes in their accounts.

### DNS Setup

Once access is granted, users can configure and set up the DNS domains required to send email campaigns from their accounts. To know more, kindly read our [**DNS Setup**](https://galaxy.maropost.com/s/article/DNS-Setup) page.

### FTP Upload

With this privilege, the user will be able to import contacts using FTP upload.

> **Note:** **Note:** When enabled, the **SFTP Access** tab is available on the Connections page.
