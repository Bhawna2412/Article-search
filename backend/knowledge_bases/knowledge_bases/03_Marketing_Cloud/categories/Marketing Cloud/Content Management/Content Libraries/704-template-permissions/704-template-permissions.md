# Template Permissions

**Template Permissions** help ensure consistent brand standards by controlling who can modify your approved email templates, protecting your design integrity while enabling appropriate team collaboration. This governance approach prevents unauthorized changes to brand-critical elements while allowing designated team members to customize content as needed.

When properly configured, template permissions can enhance workflow efficiency by establishing clear roles for template creation versus content development, ensuring your email campaigns maintain professional standards while allowing necessary flexibility for campaign customization.

You can manage the editorial permissions for your templates, restricting users from making changes to the template.

To learn more about the template library, refer to our [**Template Library**](https://galaxy.maropost.com/s/article/Template-Library) article.

## Templates Created Using the WYSIWYG Content Editor

In the WYSIWYG template editor, click the **Permissions** tab. You’ll see a list of all the users in your account. By default, these users only have viewing permissions; hence, they cannot make changes to the template. However, if you want to allow only a specific group of people to edit your template, or even restrict it to yourself, then under the **Actions** section, click on **Modify Permission** next to their names.

![template6.png](https://us.v-cdn.net/6038474/uploads/attachments/migrated/image.05TJ1000003HIPFMA4.jpg)

> **Note:** **Note:** You cannot modify the permission of a user who is an owner.

Give the user editing access and click on **Done**.

![template7.png](https://us.v-cdn.net/6038474/uploads/attachments/migrated/image.05TJ1000003H9WOMA0.jpg)

Once you’ve granted edit access to specific users, they will be able to edit your template.

> **Note:** **Note:** Template Editor permissions only apply to those who can edit the template. When email content is created using a template in the WYSIWYG Content Editor or the HTML Source Code Content Editor, users can make changes to the email content regardless of whether or not they have the rights to edit the template.

## Templates Created Using the Drag and Drop Content Editor

Templates created using the drag-and-drop content editor have an additional level of access rights. Such access control allows users to modify the template according to their preferences. This extra level of access control is ideal for enhancing the efficiency of email content production.

For example, a template can be customized so that users can only change the editorial content and images, but not the layout or styles used in the email.

By default, users of the template can only view the Drag and Drop Content Editor. However, once you’ve set permissions, users of the template can make the changes in the email content that you allow. You can access the permissions section by clicking on the **Permissions** tab in the Drag and Drop template editor.

![template8.png](https://us.v-cdn.net/6038474/uploads/attachments/migrated/image.05TJ1000003HIlfMAG.jpg)

> **Note:** **Note:** Access rights in the Drag and Drop Content Editor are defined at the time at which the content is created. If the template’s user permissions are later changed, they will not be reflected in any of the previous email content that was created using that template. The permissions will only be reflected in new email content created after the change.
