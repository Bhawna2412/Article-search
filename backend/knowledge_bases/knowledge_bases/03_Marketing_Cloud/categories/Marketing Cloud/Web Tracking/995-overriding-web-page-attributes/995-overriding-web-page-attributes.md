# Overriding Web Page Attributes

You can configure web page attribute overrides to capture more detailed campaign analytics and improve performance tracking accuracy. This enables data-driven decisions by providing complete URL information, including query parameters that influence page content, which significantly improves your ability to measure marketing effectiveness and optimize future campaigns.

When properly configured, this feature helps ensure comprehensive tracking of how visitors navigate your website through different marketing channels. This enhanced visibility into customer behavior patterns typically leads to better campaign optimization and more precise ROI measurement for your email marketing efforts.

The Page tab on the Script Builder enables you to **override the default URL** of the page being viewed. You can also override the Page Title and the URL of the referring page if you wish.

By default, Maropost's web tracking script only stores the domain and path of the web page being viewed. Query string parameters appended to the page URL are not stored.

However, if you are using query string parameters to control the actual page that is being viewed, then you will certainly want to override the default behaviour.
